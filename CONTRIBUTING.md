# 贡献指南

感谢你对 ClawPlay 的关注！我们欢迎所有形式的贡献。

## 🎯 贡献类型

### 1. 原创人格配置
分享你自己调教的 SOUL.md 人格配置。

### 2. 翻译优质人格
将英文社区的优秀人格翻译成中文（需遵守版权）。

### 3. 改进现有人格
优化已有的人格配置，提升使用体验。

### 4. 文档和教程
分享人格调教经验、使用技巧。

## 📝 提交流程

### 步骤 1：Fork 仓库
点击右上角的 Fork 按钮，将仓库复制到你的账号下。

### 步骤 2：创建分支
```bash
git checkout -b add-my-soul
```

### 步骤 3：添加你的人格配置

根据类型选择目录：
- `community/work/` - 工作助手
- `community/dev/` - 开发专家
- `community/creative/` - 创意写作
- `community/learning/` - 学习伙伴
- `translated/` - 翻译内容

创建文件，使用 kebab-case 命名：
```
community/work/agile-pm.md
```

### 步骤 4：填写内容

使用以下模板：

```markdown
# 人格名称

## 简介
一句话描述这个人格的核心特点

## 适用场景
- 具体场景1
- 具体场景2
- 具体场景3

## 特色功能
- 这个人格的独特能力1
- 这个人格的独特能力2

## 使用建议
给用户的使用提示和注意事项

## 作者信息
- 作者：你的名字或 GitHub ID
- 来源：原创 / 翻译自 [原链接]
- 协议：MIT / CC BY 4.0
- 创建日期：2026-03-06

---

## SOUL.md 内容

\`\`\`markdown
# 这里是完整的 SOUL.md 配置内容
# 用户可以直接复制使用

<identity>
你是一个...
</identity>

<personality>
...
</personality>

# 其他配置...
\`\`\`
```

### 步骤 5：更新索引

在对应分类的 README.md 中添加你的人格链接。

### 步骤 6：提交 PR

```bash
git add .
git commit -m "Add: [人格名称] - 简短描述"
git push origin add-my-soul
```

然后在 GitHub 上创建 Pull Request。

## ✅ 质量标准

### 原创内容
- ✅ 有明确的使用场景和目标用户
- ✅ 人格特征清晰，不是简单的系统提示词
- ✅ 经过实际测试，效果良好
- ✅ 中文表达流畅自然

### 翻译内容
- ✅ 必须标注原作者和原链接
- ✅ 检查原项目的开源协议
- ✅ 翻译准确，符合中文表达习惯
- ✅ 保留原作者的署名信息

### 禁止内容
- ❌ 违反法律法规的内容
- ❌ 暴力、色情、歧视性内容
- ❌ 侵犯他人版权的内容
- ❌ 恶意或有害的人格配置
- ❌ 商业化的付费内容（未经授权）

## 📜 版权和协议

### 原创内容
- 建议使用 **MIT** 或 **CC BY 4.0** 协议
- 你保留原创内容的版权
- 允许他人自由使用和修改

### 翻译内容
- 必须遵守原项目的开源协议
- 常见协议：MIT、Apache 2.0、CC BY 4.0
- 如果原项目没有明确协议，请先联系原作者

### 协议说明

**MIT 协议**：
- 允许商业使用
- 允许修改和分发
- 需保留版权声明

**CC BY 4.0 协议**：
- 允许商业使用
- 允许修改和分发
- 需署名原作者

## 🔍 审核流程

1. **自动检查**：格式、文件命名
2. **内容审核**：质量、版权、合规性
3. **测试验证**：实际效果测试（可选）
4. **合并发布**：通过后合并到主分支

通常在 1-3 天内完成审核。

## 💡 最佳实践

### 人格设计建议
1. **明确定位**：清楚这个人格解决什么问题
2. **个性鲜明**：有独特的说话风格和行为模式
3. **实用为主**：不要过度追求"有趣"而忽略实用性
4. **适度约束**：给 AI 明确的边界和行为准则

### 文档撰写建议
1. **场景具体**：不要写"适合所有人"，要写具体场景
2. **示例丰富**：提供实际对话示例更有说服力
3. **注意事项**：说明这个人格的局限性
4. **使用技巧**：分享你的调教经验
