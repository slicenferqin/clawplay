# ClawPlay Low-fi Wireframes

## 视觉方向

不要做成另一个 souls.directory 的壳。建议方向：

- **内容优先**：像工具站，不像花哨社区
- **中文语境**：标签、分类、示例和安装提示都本地化
- **可信感**：减少玩具感，强调可试用、可回滚、可判断
- **轻品牌感**：保留 ClawPlay 的人格感，但不喧宾夺主

## 关键组件

### 1. Soul Card

卡片必须在不点进去之前就回答 4 个问题：

1. 这是什么风格。
2. 适合什么场景。
3. 和什么模型兼容。
4. 是否值得我点进去。

建议字段：

- 标题
- 一句话摘要
- 标签 3-4 个
- 适用场景 2 个
- 兼容模型
- CTA: `查看详情`

### 2. Preview Card

这是 ClawPlay 的关键差异化模块。

建议信息：

- `一句话 hook`
- `适合谁`
- `风格标签`
- `典型任务`
- `预期回答感受`

### 3. Install Box

安装框必须非常明确，避免用户读完还不知道怎么装。

包含：

- CLI 命令
- 手动安装入口
- 备份提示
- 回滚提示

## 页面交互建议

- 搜索框支持即时筛选
- 卡片 hover 时只做轻微强调，不做重动画
- `复制安装命令` 成功后显示明确反馈
- 示例对话默认展开 1-2 组，避免页面太长
- 原始 SOUL 内容默认折叠

## 内容优先级

详情页上的信息优先级：

1. `这是什么`
2. `适合谁`
3. `装上去会怎样`
4. `怎么安装`
5. `原始内容长什么样`

如果页面空间不够，先保证这 5 件事完整，再谈美化。

## 下一步如何接代码

原型图完成后，前端实现建议按这个顺序：

1. 全局 Layout 和 Header
2. 首页 Hero 和 Soul Card
3. 列表页搜索与筛选
4. 详情页 Preview Card + Install Box
5. `api/raw/[slug]`
6. 移动端收口

这样能最快做出一个可用版本，而不是先堆漂亮样式。
