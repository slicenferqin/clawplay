## 1. 背景

P1 到 P5 已经把 ClawPlay 的主链路、审核效率、SEO、可信度和最小运维能力基本补齐。

接下来进入 P6，不应该直接上复杂增长系统，而应该先做**低成本、高确定性**的承接入口：

- 新用户第一次来，不知道从哪挑时，有没有明确入口
- 开发者用户、普通用户、想尝鲜的人，能不能快速走到适合自己的第一批 Soul
- 首页与导航里，“合集”到底是不是一个真正可传播、可复用的页面，而不是一个占位词

## 2. 目标

本阶段目标：

1. 把“合集”从导航文案升级成真正页面。
2. 提供 3-4 组低认知成本的推荐入口。
3. 让首页、灵魂库页和全站导航都能把用户导向这些入口。
4. 尽量复用现有数据，不引入复杂推荐系统。

## 3. 非目标

本阶段不做：

- 个性化推荐算法
- 登录态推荐
- 社区排行榜系统
- 单独的营销 CMS
- 标签自由组合生成动态落地页

## 4. 交付范围

1. `/collections` 合集页
2. 四组首批增长入口：
   - 新手首选
   - 开发首选
   - 当前热门
   - 最近新增
3. 首页 / 灵魂库页 / 导航 / 页脚的合集入口回填
4. SEO 索引与 smoke check 更新
5. 一份实施记录文档

## 5. 方案设计

### 5.1 入口要“能直接选”，不是“继续想”

合集页的第一原则不是展示更多内容，而是替用户减少第一次选择成本。

所以第一刀采用固定入口，而不是开放筛选组合：

- 新手首选：给第一次接触 ClawPlay 的人
- 开发首选：给明显偏工程 / 开发工作流的人
- 当前热门：给想直接看“大家最近更爱装什么”的人
- 最近新增：给想尝鲜、看更新的人

### 5.2 热门入口复用现有行为信号

当前已经有轻量埋点与热门榜能力，所以“当前热门”直接复用：

- 详情查看
- Hero 预览点击
- 安装复制
- 原文下载 / 复制

如果真实行为样本暂时不足，则使用精选 Soul 回填，保证页面始终可用。

### 5.3 其它入口保持人工策展

“新手首选”和“开发首选”第一刀继续用人工策展：

- 解释成本更低
- 输出更稳定
- 更符合冷启动阶段的内容密度

后续再考虑升级成规则化推荐。

## 6. 实施拆解

1. 产出合集页数据 helper
2. 新增 `/collections` 页面
3. 回填导航与首页入口
4. 更新 sitemap / robots / smoke check
5. 补实施文档与 README 路线图状态

## 7. 验收标准

满足以下条件即可视为完成：

1. `/collections` 可访问，内容为中文且具备清晰入口分组
2. 热门入口可基于现有行为信号返回结果
3. 新手首选 / 开发首选 / 最近新增三组内容稳定可用
4. 站内顶部导航与页脚都能进入合集页
5. 首页或灵魂库页至少有一个明显的合集导流入口
6. 新页面已进入 sitemap / robots / smoke check

## 8. 上线与回滚

### 上线顺序

1. 先发合集 helper 和页面
2. 再发导航与首页导流
3. 最后发 SEO / smoke check / README 文档同步

### 回滚策略

- 合集页与导流入口均为增量能力，可独立回滚
- 热门入口若行为样本异常，可暂时只保留精选回填
- 不影响投稿、安装和审核主链路
