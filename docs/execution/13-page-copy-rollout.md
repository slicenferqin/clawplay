# 13 Page Copy Rollout

- Status: Ready
- Depends On: `docs/execution/11-prelaunch-content-assets.md`
- Owner: Codex + slicenfer
- Last Updated: 2026-03-09

## 1. 背景

`docs/content/prelaunch-asset-pack.md` 已经完成，ClawPlay 的对外主叙事、推荐 Soul 话术、安装引导和投稿引导都已经有统一母稿。

但这些内容目前还主要停留在文档层，没有真正回填到首页、安装页、投稿页。

这会带来一个明显问题：

- 文档里已经能讲清楚
- 页面里还没有完全讲清楚

所以 P4 的下一刀，不再是继续产出更多文案，而是把已经确定的内容资产真正落实到页面承接链路里。

## 2. 目标

本阶段目标：

1. 让首页、安装页、投稿页使用同一套主叙事。
2. 强化“可浏览、可比较、可安装”的页面承接。
3. 降低新用户第一次安装和第一次投稿的心理门槛。
4. 不破坏现有交互骨架，优先做高价值文案与结构增强。

## 3. 非目标

本阶段不做：

- 大改首页视觉结构
- 重做安装流程交互
- 重做投稿表单字段结构
- 新增复杂 onboarding 流程

## 4. 交付范围

本阶段交付：

1. 首页文案与承接区更新
2. 安装页文案与 FAQ 更新
3. 投稿页与投稿表单引导文案更新
4. 对应实施记录文档

## 5. 方案设计

### 5.1 首页

首页重点补三件事：

- Hero 下方新增更明确的三步价值说明
- 强化 CTA，让“去浏览”“去安装”“去投稿”更清晰
- 把安装条改成更像行动引导，而不是一句轻提示

### 5.2 安装页

安装页重点补：

- 装前先备份
- 先看预览再安装
- 不喜欢可回滚
- 常见问题口径

### 5.3 投稿页

投稿页重点补：

- 强化“先交首稿，不求一次填满”的低门槛感
- 强化“.md 文件直传”优势
- 让来源、授权、审核规则表达得更可信、更清楚

## 6. 验收标准

满足以下条件即完成：

- 首页、安装页、投稿页都能映射到统一主叙事
- 页面文案能直接复用资产包中的核心表达
- 新用户第一次进入页面时，不需要额外解释也能理解接下来该做什么
