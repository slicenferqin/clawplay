# 19 Collection Share Copy

- Status: Ready
- Depends On: `docs/execution/18-shareable-collection-pages.md`
- Owner: Codex + slicenfer
- Last Updated: 2026-03-09

## 1. 背景

P6 第二刀已经让每个合集拥有独立专题页与分享图，但传播层还差最后一小步：

- 页面能分享，不代表用户愿意马上分享
- 即使用户想发，也往往还要自己组织推荐文案
- 传播动作多一步，就会明显降低真实扩散率

所以第三刀不该继续先做更复杂的增长系统，而是把“能发出去”这件事做得更顺手。

## 2. 目标

本阶段目标：

1. 给每个专题页补可直接复制的外发文案模板。
2. 让专题页本身更像可传播资产，而不只是站内内容页。
3. 继续为域名放开后的真实扩散提前铺底。

## 3. 非目标

本阶段不做：

- 社交平台一键分享 SDK 接入
- 自动生成几十种渠道定制文案
- 增长后台或分享效果归因看板

## 4. 交付范围

1. 专题页“分享这组”区块
2. 专题链接复制入口
3. 每个合集 2 条外发文案模板
4. smoke check 回填一条文案区验证
5. 一份实施记录文档

## 5. 方案设计

### 5.1 不加复杂分享控件，先把复制动作做好

当前最有效的动作不是接一堆外部平台 SDK，而是先把：

- 专题链接
- 分享图入口
- 可复制文案

放在同一个区块里，让用户无脑拿走。

### 5.2 每个合集至少提供两档文案

每个合集先提供：

- 短文案：适合发群、私聊、评论区
- 长文案：适合发帖、写推荐说明

### 5.3 文案写法强调“为什么值得点开”

模板不只是重复标题，而要直接回答：

- 这组适合谁
- 为什么这组有价值
- 点进去会得到什么

## 6. 验收标准

1. `/collections/starter` 出现“分享这组”区块
2. 至少能复制专题链接和两条模板文案
3. 分享图入口可打开
4. smoke check 能检查到该区块文案
