# 09 Admin Review Efficiency Implementation

- Status: Done
- Depends On: `docs/execution/09-admin-review-efficiency.md`
- Owner: Codex + slicenfer
- Last Updated: 2026-03-09

## 1. 本次实施范围

本次完成了 P2 的第一刀，把后台审核从“普通列表 + 详情页”升级成“带优先级判断的审核队列”。

具体包括：

1. 抽出共享审核队列评估逻辑
2. 新增快捷视图：缺来源 / 阻断项 / 待发布
3. 列表页改成卡片化审核队列
4. 详情页补上审核优先级卡片

## 2. 关键实现

### 2.1 共享审核优先级模块

新增：`src/lib/submissions/review-queue.ts`

负责统一计算：

- 来源状态
- 是否缺来源
- 是否缺原作者
- 阻断 / 警告 / 通过数量
- 审核优先级标签
- 下一步建议
- 风险摘要

这样列表页与详情页不会再各自维护一套判断逻辑。

### 2.2 列表页升级为审核队列

修改：`src/app/admin/submissions/page.tsx`

主要变化：

- 增加 4 个快捷视图入口
- 列表默认按审核优先级排序
- 原表格改为审核卡片列表
- 每条卡片直接展示：
  - 状态
  - 优先级
  - 来源状态
  - 风险摘要
  - 下一步建议
  - 阻断 / 警告 / 通过计数

### 2.3 详情页补上优先级展示

修改：`src/app/admin/submissions/[id]/page.tsx`

新增顶部“审核优先级”卡片，让管理员进入详情页后依然保留同一套判断上下文。

### 2.4 服务层补充全量读取接口

修改：`src/lib/submissions/service.ts`

新增 `listAllSubmissions()`，用于后台审核队列在服务端做优先级分析与快捷视图聚合。

### 2.5 样式补充

修改：`src/app/globals.css`

新增后台审核队列样式，包括：

- 快捷视图卡片
- 审核卡片列表
- 风险块与统计胶囊
- 详情页优先级卡片态

## 3. 验证方式

本次主要采用两类验证：

1. `npm run build`
2. 本地启动后回归后台登录页、审核列表页、详情页

## 4. 结果

这次改动后，后台列表不再只是“能看投稿”，而是已经具备第一版工作台属性：

- 管理员可以快速切出最值得优先处理的稿件
- 列表页即可完成第一轮风险分诊
- 详情页与列表页的审核语言保持一致

这也为下一步继续做“审核效率第二刀”打下了统一的判断基础。
