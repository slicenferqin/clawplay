## 1. 关键决策收口

### 1.1 先出资产包，再决定哪些文案回填页面

原因：

- 现在备案未放开，最值钱的是先把传播资产准备好
- 如果先改页面，很容易在没有统一口径的情况下边改边想
- 先有资产包，后续不论改首页、安装页还是对外发帖，都有统一母稿可复用

### 1.2 首批只做“高频使用素材”

本次先覆盖四类：

1. 站点定位与对外介绍
2. 代表 Soul 推荐卡文案
3. 安装 / 投稿 / FAQ 说明
4. 社交平台传播模板

本次不做：

- 品牌长故事
- 新闻稿体正式 PR
- 平台专属视觉海报文案体系

### 1.3 推荐 Soul 先固定为 5 个

本次资产包采用固定 5 个代表 Soul：

- `edict-counselor`
- `code-reviewer`
- `grumpy-wang`
- `catgirl-nova`
- `architect`

选择原因：

- 覆盖规划、开发、审查、创意、架构五类高频场景
- 与首页 Hero 轮播和精选内容有较高一致性
- 适合做首波传播时的“差异化样本”

## 3. 资产包结构

`docs/content/prelaunch-asset-pack.md` 统一包含：

1. 一句话定位
2. 长短版项目介绍
3. 首页核心文案候选
4. 5 个代表 Soul 的推荐文案
5. 安装引导文案
6. 投稿引导文案
7. FAQ 标准回答
8. 社交传播模板

这样可以确保：

- 对外说法一致
- 页面回填有依据
- 后续做发布日素材时不需要重新拼口径

## 4. 口径约束

### 4.1 必须强调的价值点

所有文案尽量围绕：

- 可浏览
- 可比较
- 可安装

### 4.2 尽量少讲的词

为了避免当前阶段显得过重，以下说法尽量少用：

- 交易平台
- Soul 市场
- 生态基础设施
- 开放社区治理平台

当前更适合的定位，是一个中文 Soul 目录与分享站。

### 4.3 术语统一

统一使用：

- Soul / 灵魂
- 安装
- 投稿
- 预览
- 原文
- 标签

避免一会儿叫模板、一会儿叫人格、一会儿又叫角色配置，减少认知摇摆。

## 5. 后续接入顺序

如果下一步要把资产包回填到页面，建议顺序：

1. 首页 Hero 与安装引导短文案
2. 投稿页顶部说明与低门槛提示
3. 安装页步骤说明与 FAQ
4. Soul 详情页可选补一条“适合谁”的摘要句

这样做的原因是：首页、安装页、投稿页最直接影响转化。

## 6. 验证路径

本次文档交付完成后，最少检查：

1. 是否覆盖路线图要求的核心内容资产
2. 推荐 Soul 是否与当前站内内容一致
3. 文案口径是否能映射到首页现有叙事
4. 是否足够短、足够能直接拿去发
