# ClawPlay Soul Metadata 规范（MVP）

- Last Updated: 2026-03-08
- Scope: 新投稿、后台审核、发布前检查

## 1. 目标

这份规范用于回答三个问题：

1. 一份 Soul 最少要写到什么程度，才值得进入 ClawPlay。
2. 哪些字段只是建议优化，哪些字段会直接阻断发布。
3. 投稿端、审核端和最终发布端如何共用同一套规则。

## 2. 字段分层

### 2.1 阻断发布的核心字段

以下字段若不达标，不允许直接发布：

- `previewHook`
- `previewPrompt`
- `previewResponse`
- `rawSoul`
- `license`
- `rightsStatement`
- `sourceUrl`（仅翻译 / 改编）

### 2.2 建议优化的展示字段

以下字段不会阻断投稿，但会影响审核体验、列表页转化和 SEO：

- `title`
- `summary`
- `tags`
- `features`
- `suggestions`
- `sourceAuthor`

## 3. 推荐范围

### 3.1 标题

- 建议长度：4-28 个字符
- 目标：列表页一眼看懂角色定位
- 避免：过长前后缀、纯梗图式命名、模糊口号

### 3.2 摘要

- 建议长度：24-100 个字符
- 目标：一句话说清“适合做什么”
- 避免：只有“很强”“很懂你”这类空泛褒义

### 3.3 标签

- 建议数量：3-6 个
- 至少 2 个标签应描述能力或任务场景
- 优先复用站内推荐词表，减少同义词分裂

### 3.4 预览三件套

- `previewHook`：至少 12 字
- `previewPrompt`：至少 10 字
- `previewResponse`：至少 24 字
- 原则：必须能独立展示出风格、能力和使用场景

### 3.5 原始 SOUL

- 最低发布门槛：80 字
- 原则：尽量提供接近最终安装态的 `SOUL.md`

### 3.6 权利与来源

- `license` 不能是“待确认 / TBD / 未知”一类占位值
- `rightsStatement` 建议至少 12 字
- 翻译 / 改编必须提供 `sourceUrl`
- 翻译 / 改编建议提供 `sourceAuthor`；未知时应明确说明

## 4. 投稿类型规则

### 4.1 原创

至少要求：

- 作者署名
- 清晰协议
- 投稿权利声明

### 4.2 翻译

至少要求：

- 原始来源链接
- 原作者署名或“未知原作者”说明
- 当前译稿的协议与可发布说明

### 4.3 改编

至少要求：

- 原始来源链接
- 原作者署名或来源说明
- 当前改编范围 / 权利说明

## 5. 审核结论分级

### Blocking

会直接阻断发布：

- 预览三件套过短
- 原始 SOUL 过短
- 协议仍是占位值
- 权利声明过短
- 翻译 / 改编缺 `sourceUrl`

### Warning

不会阻断发布，但应在审核时优先优化：

- 标题过短或过长
- 摘要过短或过长
- 标签数量不合适
- 标签未复用推荐词表
- 亮点 / 使用建议过少
- 翻译 / 改编缺 `sourceAuthor`

## 6. 执行原则

- 投稿前台以“提示”为主，不因为推荐值不达标而阻止提交
- 后台审核以“最低发布规范”为准，守住质量与合规底线
- 旧数据渐进清理，不要求一次性全部回填

## 7. 当前待决议项

以下问题仍保留为后续架构决策，不在本规范内强行落地：

- `translated` 是否继续保留为一级分类
- 是否把翻译来源完全收敛为 `sourceType` + `sourceUrl` 维度
